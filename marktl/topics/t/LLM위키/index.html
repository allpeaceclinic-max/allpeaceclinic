LLM 위키 (LLM Wiki)

개념 정의

앤드레이카파시가 제안한 개인 지식베이스 관리 패턴. LLM이 점진적으로 마크다운 위키를 구축·유지하는 시스템으로, 전통적인 RAG(검색 후 생성)와 달리 지식을 한 번 컴파일하고 지속적으로 갱신한다.

이 Vault에서의 구현

이 0doctorvalut이 바로 LLM Wiki 패턴의 실제 구현체다.

카파시 개념 이 Vault
Raw Sources raw/ 폴더 (절대 수정 안 함)
The Wiki wiki/ 폴더 (LLM이 생성·유지)
The Schema CLAUDE.md
Ingest 새 raw 파일 → 위키 페이지 업데이트
Query wiki 탐색 → 합성 답변
Lint 모순·고아 페이지·누락 주제 점검

3계층 구조

Raw Sources (불변) ↓ LLM reads only The Wiki (LLM 소유) ↑ governed by The Schema (CLAUDE.md)

핵심 Operations

Ingest

  1. raw/에 원본 저장
  2. 관련 위키 페이지 파악
  3. 합성·요약 작성 및 업데이트
  4. index.md 갱신
  5. log.md에 기록

Query

  1. 관련 wiki 페이지 탐색
  2. 인용과 함께 합성 답변
  3. 가치 있는 답변 → 새 페이지로 보존

Lint

  • 모순 탐지
  • 고아 페이지 파악
  • 미생성 주제 제안
  • 오래된 프로젝트 상태 확인

왜 LLM이 적합한가

인간은 교차 참조 업데이트, 일관성 유지, 모순 지적 같은 반복적이고 지루한 지식 유지 작업에서 쉽게 지친다. LLM은 피로 없이 이 작업을 수행하므로, 지식이 흩어지지 않고 복리로 축적된다.
